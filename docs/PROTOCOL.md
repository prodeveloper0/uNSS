# 클라이언트 ↔ 서버 계약 사항

클라이언트와 서버는 코드를 공유하지 않고 HTTP 로만 통신합니다. 두 쪽 중 한 쪽을 수정할 때 아래 계약을 반드시 확인하세요.

## 사용자 식별

- 클라이언트는 Switch 계정의 **nickname** 을 `user_name` 경로 파라미터로 사용합니다 (uid 아님).
- 따라서 Switch 사용자 이름을 변경하면 서버에 저장된 세이브 데이터가 고아가 됩니다.

## 업로드 포맷

- 업로드는 `application/octet-stream` 본문으로 raw 스트리밍됩니다.
- 본문은 `zipio` 가 만든 zip 아카이브이지만, 서버는 내용을 검사하지 않고 그대로 `savedata/<revision_id>` 에 씁니다.
- 즉, zip 포맷 자체는 서버와 무관한 클라이언트 사이드 규약입니다.

## 응답 포맷

- 서버 응답은 **plain text** 이며 JSON 이 아닙니다.
- C++ 클라이언트 (`remote.cpp` / `http.cpp`) 가 이 텍스트를 직접 파싱합니다.
- 서버 응답을 JSON 으로 "개선" 하고 싶다면 반드시 클라이언트 파서도 같이 업데이트해야 합니다.
